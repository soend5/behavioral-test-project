# V2.1 助教效能 需求文档

## 概述
本次迭代为助教效能阶段（2周），聚焦客户筛选、高风险预警、批量操作、跟进模板等功能，大幅提升助教运营效率。

**核心目标**：
- 助教筛选效率 +50%
- 高风险用户响应率 +30%
- 移动端助教体验优化

---

## 功能模块

### 1. 客户筛选器

#### 1.1 筛选功能
- **现状**: 客户列表无筛选，只能滚动查找
- **需求**: 支持多维度筛选
- **筛选维度**:
  - 分层：高潜力 / 需关注 / 活跃 / 沉默 / 新客户
  - 画像：6种画像类型
  - 阶段：pre / mid / post
  - 最近活动：7天内 / 14天内 / 30天内 / 更早
- **终局模型支持**: 是（阶段：全陪跑阶段；标签逻辑：分层/画像筛选）

#### 1.2 UI 设计
- Web端：侧边栏筛选
- 移动端：底部抽屉式筛选

#### 1.3 验收标准
- [ ] 筛选器 UI 可用
- [ ] 支持分层、画像、阶段、活动时间筛选
- [ ] 筛选结果实时更新
- [ ] 移动端抽屉式筛选可用

---

### 2. 高风险用户预警

#### 2.1 预警标识
- **需求**: 为高风险客户提供视觉预警
- **实现**:
  - 客户列表：红色边框 + ⚠️图标
  - 客户详情：进入时弹出风险提示
  - 待办面板：高风险用户置顶
- **终局模型支持**: 是（阶段：行动推进期；标签逻辑：高风险标记）

#### 2.2 高风险判定条件
- 分层为"需关注"
- 画像为"情绪敏感型"相关
- 超过14天无互动

#### 2.3 验收标准
- [ ] 高风险用户有红色边框
- [ ] 详情页显示风险提示
- [ ] 待办面板高风险置顶

---

### 3. 批量操作功能

#### 3.1 批量操作
- **需求**: 支持对多位客户执行批量操作
- **功能**:
  - 批量标记已处理
  - 批量添加标签
  - 批量创建跟进记录
- **终局模型支持**: 否

#### 3.2 风险控制
- 初版限制小批量（最多50个客户）
- 操作需二次确认
- 保证原子性

#### 3.3 验收标准
- [ ] 批量选择 UI 可用
- [ ] 批量操作 API 可用
- [ ] 操作有确认提示
- [ ] 限制批量数量

---

### 4. 跟进记录模板

#### 4.1 模板功能
- **需求**: 提供标准化跟进记录模板
- **模板类型**:
  - 首次沟通
  - 跟进沟通
  - 训练提醒
  - 复测邀请
- **终局模型支持**: 是（阶段：成果巩固期）

#### 4.2 验收标准
- [ ] 模板选择 UI 可用
- [ ] 选择模板后自动填充内容
- [ ] 支持编辑后保存

---

### 5. 移动端助教体验优化

#### 5.1 底部导航栏
- **需求**: 移动端添加底部导航栏
- **导航项**: 仪表盘、客户、消息、我的

#### 5.2 验收标准
- [ ] 移动端显示底部导航
- [ ] 导航切换正常
- [ ] 关键操作按钮足够大
