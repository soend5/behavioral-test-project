# V1.3 UI/UX 增强需求文档

## 概述
本次迭代包含四个主要功能模块，旨在提升测评结果展示、邀请状态页面、助教工作台通知、以及管理后台配置整合。

---

## 功能模块

### 1. 测评结果页增强 (Result Page Enhancements)

#### 1.1 原型展示 (Archetype Display)
- **需求**: 在结果页显示用户的原型（archetype）信息
- **数据来源**: `archetypes_v1.json` 中的 `title_cn`, `one_liner_cn`, `self_recognition_cn`
- **展示内容**:
  - 原型标题（如"规则执行型"）
  - 一句话摘要（one_liner_cn）
  - 自我识别点（self_recognition_cn，可选展示 1-2 条）

#### 1.2 一句话摘要修复
- **现状**: 当前摘要过于通用，缺乏个性化
- **需求**: 根据原型和行为标签生成更具针对性的摘要

#### 1.3 更多维度结果展示
- **需求**: 展示更多行为维度的结果（rule, risk, emotion, consistency, opportunity, experience）
- **展示方式**: 使用 `lib/tag-display.ts` 中的 `BEHAVIOR_GROUP_DISPLAY` 数据

#### 1.4 助教二维码展示
- **需求**: 在结果页底部展示助教的微信二维码，方便用户联系
- **数据库变更**: User 表需新增字段
  - `name` (String?): 助教真实姓名/显示名
  - `wechat_qrcode` (String?): 微信二维码图片 URL
- **展示逻辑**: 如果助教有配置二维码则显示，否则显示默认联系方式

---

### 2. 邀请状态页 V1.3 UI (Invite Status Pages)

#### 2.1 已完成状态 (completed)
- **现状**: 简单提示"本次测评已完成"
- **需求**: 
  - 更友好的完成提示
  - 突出"查看结果"和"联系助教"两个行动点
  - 添加鼓励性文案

#### 2.2 已过期状态 (expired)
- **现状**: 简单提示"邀请已失效"
- **需求**:
  - 解释过期原因（时间过期）
  - 提供明确的下一步指引（联系助教获取新链接）

#### 2.3 已关闭状态 (closed) - 如果存在
- **需求**: 类似过期状态的处理

---

### 3. 助教工作台通知 (Coach Dashboard Notifications)

#### 3.1 测评完成通知
- **需求**: 当参与者完成测评时，助教工作台显示通知/弹窗
- **实现方式**: 
  - 轮询检查新完成的测评（简单方案）
  - 或使用 WebSocket（复杂方案，暂不实现）
- **通知内容**: 参与者昵称、完成时间、快捷查看链接

#### 3.2 CRM 风格待办区域
- **需求**: 在工作台添加"待处理"区域
- **待办类型**:
  - 新完成的测评（待查看结果）
  - 进行中的测评（已开始但未完成）
  - 即将过期的邀请
- **展示**: 按优先级排序，支持快捷操作

---

### 4. 管理后台题库设置整合 (Admin Quiz Settings Consolidation)

#### 4.1 移动默认题库版本配置
- **现状**: 默认题库版本配置在 `/admin/settings` 页面
- **需求**: 将此配置移动到 `/admin/quiz` 题库管理页面
- **原因**: 配置与题库强相关，放在一起更直观

#### 4.2 设置页面调整
- **需求**: 从设置页面移除题库版本配置，或保留为只读展示

