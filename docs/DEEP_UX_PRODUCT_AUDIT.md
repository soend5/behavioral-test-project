## 0）执行摘要

### 项目一句话定位（优化版）

**「帮散户看清自己的交易行为结构，帮助教用对的节奏陪跑——不荐股、不预测、只做行为镜子与沟通桥梁」**

原定位"交易行为结构测评与助教陪跑工具"过于功能化，建议改为更有价值感的表达。

### 最大亮点 TOP3

| 排序 | 亮点 | 说明 |
|------|------|------|
| 1 | **合规边界清晰** | 从数据模型到UI文案，全链路不存储收益数据、不输出投资建议，合规护栏设计到位 |
| 2 | **测评题目"人话化"程度高** | 9题快测用散户真实场景描述，选项不带对错暗示，降低理解门槛 |
| 3 | **SOP匹配引擎可配置** | 标签+阶段→策略的规则引擎支持Admin配置，为后续个性化陪跑留出空间 |

### 最大风险 TOP3

| 排序 | 风险 | 影响 | 紧急度 |
|------|------|------|--------|
| 1 | **Aha Moment 不清晰** | 用户完成测评后"然后呢？"感强烈，价值感知断层 | 🔴 高 |
| 2 | **结果页信息密度过高** | 维度/标签/画像堆叠，散户看不懂、记不住、不知道下一步 | 🔴 高 |
| 3 | **助教工具缺乏"一键话术"** | realtime_panel有策略但无可直接复制的话术，转化效率受限 | 🟡 中 |

## 1）四角色体验报告

### A. 客户视角（散户/小白/进阶）

#### 关键目标与关键任务

| 用户类型 | 核心目标 | 关键任务 |
|----------|----------|----------|
| 小白散户 | 知道自己炒股为什么总亏 | 完成测评→看懂结果→知道下一步 |
| 进阶用户 | 验证/优化自己的交易系统 | 深度测评→对比历史→找到改进点 |
| 观望用户 | 评估这个服务是否值得信任 | 快速体验→感受专业度→决定是否深入 |

#### 端到端体验路径

```
1. 收到邀请链接（微信/短信）
2. 点击进入落地页 /t/[token]
3. 阅读邀请信息+合规提示
4. 点击"开始测评"
5. 进入测评页 /t/[token]/quiz
6. 逐题作答（9题/18题）
7. 点击"提交测评"
8. 跳转结果页 /t/[token]/result
9. 查看结果（画像/维度/标签）
10. 决定下一步（联系助教/复测/离开）
```

#### 情绪曲线分析

```
兴奋度
  ↑
  │     ┌─┐ 开始测评
  │    ╱  ╲
  │   ╱    ╲ 答题中（略疲惫）
  │  ╱      ╲     ┌─┐ 看到画像名称
  │ ╱        ╲   ╱  ╲
  │╱          ╲─╱    ╲ 看不懂维度
  │                   ╲_____ 然后呢？
  └────────────────────────────→ 时间
   落地页  测评中  提交  结果页  离开
```

**关键情绪节点：**
- ✅ 兴奋点：看到"你是XX型"的画像名称
- ⚠️ 困惑点：六维雷达图/标签堆叠看不懂
- ❌ 流失点：结果页底部"然后呢？"没有清晰行动指引

#### 阻力点清单

| 严重度 | 阻力点 | 位置 | 影响 | 改进建议 |
|--------|--------|------|------|----------|
| S1 | 结果页信息过载 | /t/[token]/result | 用户看不懂、记不住 | 重构为"一句话+一个行动" |
| S1 | 下一步行动不清晰 | 结果页底部 | 转化断层 | 增加明确CTA+助教入口 |
| S2 | 测评进度无感知 | /t/[token]/quiz | 中途放弃率高 | 增加进度条+预估时间 |
| S2 | 合规提示过于醒目 | 落地页顶部 | 信任感下降 | 调整位置和措辞 |
| S3 | 移动端选项点击区域小 | 测评页 | 误触/体验差 | 增大点击区域 |
| S3 | 无法保存进度 | 测评页 | 中断后需重来 | 支持断点续答 |

#### 改进方案（具体到页面/交互/文案）

**结果页重构方案：**

```
【当前结构】
A) 一句话摘要 → 太抽象
B) 行为维度 → 6个维度看不懂
C) 显著行为特征 → 标签堆叠
D) 下一步建议 → 太泛
E) 联系助教 → 位置太靠后

【优化结构】
1. 首屏：一句话画像 + 一个核心洞察
   "你是「规则执行型」：动手前会先把条件写清楚，再执行。"
   
2. 核心洞察（只展示1-2个最显著的）
   "你的规则依赖度偏高，这意味着..."
   
3. 下一步行动（强CTA）
   "想知道怎么用好这个特点？联系你的助教 [姓名]"
   [扫码添加微信] [查看详细报告]
```

#### 独特性与护城河评估

**独特性：**
1. **行为结构测评**：市面少见的"不预测结果、只识别结构"的测评产品
2. **SOP匹配引擎**：标签驱动的个性化陪跑策略，可配置可迭代
3. **合规边界清晰**：从产品设计到技术实现，全链路合规

**护城河：**
| 护城河类型 | 当前状态 | 建设建议 |
|------------|----------|----------|
| 数据护城河 | ⚠️ 弱 | 积累用户行为数据，优化测评模型 |
| 内容护城河 | ⚠️ 中 | 持续迭代题库+画像+SOP |
| 网络效应 | ❌ 无 | 考虑助教社区/用户社区 |
| 品牌护城河 | ❌ 无 | 建立"行为结构测评"品类认知 |

#### 产品定位与叙事建议

**当前叙事问题：**
- "交易行为结构测评"太技术化，用户不理解
- "助教陪跑"太模糊，价值感不强

**建议叙事：**

```
【对用户】
"炒股总亏？先看清自己的操作习惯"
→ 3分钟测评，看清你在哪一步容易乱动

【对助教】
"用对的节奏陪跑，让转化自然发生"
→ 基于测评结果的个性化沟通策略

【对合作方】
"合规的投教工具，可复制的陪跑模式"
→ 不荐股、不预测、只做行为镜子
```

```typescript
// 话术模板数据结构
interface ScriptLibrary {
  categories: ScriptCategory[];
}

interface ScriptCategory {
  id: string;
  name: string;           // "首次沟通"
  scripts: Script[];
}

interface Script {
  id: string;
  name: string;           // "规则执行型-首次沟通"
  trigger: ScriptTrigger;
  content: string;
  variables: string[];
  tags: string[];         // 用于搜索
  usageCount: number;     // 使用统计
  effectiveness: number;  // 效果评分
}

// 话术变量替换
function renderScript(script: Script, context: CustomerContext): string {
  let content = script.content;
  for (const variable of script.variables) {
    content = content.replace(`{{${variable}}}`, context[variable] || '');
  }
  return content;
}
```

### 适度合规模板

**问题：当前合规提示过于冗长，且多处重复**

1. 关键节点必须出现，但措辞简洁

3. 用正向表达替代否定堆叠

**优化前：**

```
本测评及结果用于生成行为结构画像与沟通建议参考，不构成投资顾问服务或任何买卖建议，不承诺收益。
```
