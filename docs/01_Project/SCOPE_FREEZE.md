# SCOPE_FREEZE

## 目的

明确 V1 灰度上线的“有效范围”（做什么/不做什么），便于验收与防止需求漂移。

## 适用范围

- V1 灰度上线（fast/pro 测评 + 结果页 + coach/admin 最小后台）

## 非目标

- 不覆盖后续版本的扩展设计

## 目录 / TODO

- [ ] V1 功能清单（Client/Coach/Admin）
- [ ] 不做清单（例如：投顾/交易信号/收益承诺/复杂画像运营）
- [ ] 数据闭环最小集（埋点/审计/复盘）
- [ ] 冻结点与变更流程（灰度/回滚/版本策略）

## 关联实现/数据位置

- 页面：`app/`
- API：`app/api/`
- 门禁：`lib/authz.ts`、`middleware.ts`
- 内容资产：`data/seed/*.json`
